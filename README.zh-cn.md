# Check Flow

VS Code 代码检查清单管理器。将文件按 **分组** 和 **阶段** 组织，用 **Todo 子项** 追踪检查进度，让每次 Review 都有条不紊。

## 功能特性

### 分组 & 阶段 —— 两级组织结构

- **分组（Group）** —— 顶层分类，如"前端检查"、"后端接口"、"QA 验收"
- **阶段（Phase）** —— 分组内的子阶段，如"第一阶段 — 核心逻辑"、"Sprint 42"

双击分组或阶段名称可直接原地重命名。

### 文件 + Todo 子项

每个文件可以挂载独立的 Todo 列表：

```
[□] UserInfo.vue                   1/3
    │ [☑] 检查移动端布局
    │ [□] 验证 API 错误状态
    │ [□] 检查 loading 骨架屏
    │ + 添加待办…
```

- 点击侧边栏工具栏中的 **≡ Todos** 按钮，切换文件列表 / 文件+Todo 两种视图
- **所有 Todo 完成** → 父级文件自动勾选为完成
- **任意 Todo 未完成** → 父级文件自动取消勾选
- 文件行右侧的 `1/3` 徽章即使在隐藏 Todo 的视图下也会显示进度

### 进度追踪

每个分组和阶段都显示 `已完成/总数` 徽章和动态进度条。

### 拖拽排序

在同一阶段内或跨阶段/跨分组拖动文件行，蓝色插入线精准指示落点。

### 从文件浏览器添加文件

**方式一 —— 右键菜单**（推荐，支持多选）

在 Explorer 中选中一个或多个文件/文件夹 → 右键 → **添加到 Check Flow…** → 选择分组 → 选择阶段。

> 支持多选：先按住 `Ctrl` / `Cmd` 选中多个文件再右键。

**方式二 —— + Files 按钮**

点击阶段名旁的 **+ Files** → 打开系统文件选择框（支持多选和文件夹）。

**方式三 —— 拖放区域**

从操作系统的文件管理器拖动文件到阶段的拖放区域。

### JSON 导出 / 导入

面板标题栏中的图标：

| 图标 | 操作 |
|------|------|
| 导出 | 将完整清单保存为可读性强的 JSON 文件 |
| 导入 | 从 JSON 文件加载（可选择替换或合并） |

导出格式对人类和 AI 都友好，适合直接粘贴给 AI 进行分析：

```json
{
  "version": 1,
  "exportedAt": "2026-03-15T10:00:00.000Z",
  "workspace": "my-project",
  "summary": "2 个分组，8 个文件，5 个已完成",
  "groups": [
    {
      "name": "前端检查",
      "phases": [
        {
          "name": "第一阶段 — 核心",
          "progress": "2/4 files",
          "files": [
            {
              "name": "index.vue",
              "path": "src/views/index.vue",
              "checked": true,
              "todos": [
                { "text": "检查布局", "checked": true },
                { "text": "检查接口", "checked": true }
              ]
            }
          ]
        }
      ]
    }
  ]
}
```

### 一键折叠

点击面板标题栏中的折叠图标（或执行 **Check Flow: 折叠 / 展开所有分组**），将所有分组一键折叠；再次点击展开全部。

## 快速上手

1. 点击活动栏中的 **检查流程** 图标打开侧边栏
2. 点击 **＋ Group** → 输入分组名称，如"前端检查"
3. 展开分组，点击 **+ Phase** → 输入阶段名称，如"Sprint 42"
4. 在 Explorer 中右键文件 → **添加到 Check Flow…** → 选择分组和阶段
5. 点击文件名即可在编辑器中打开该文件
6. 检查完成后勾选复选框，或先添加 Todo 进行更细粒度的追踪

## 快捷键

| 操作 | 按键 |
|------|------|
| 确认内联表单 | `Enter` |
| 取消表单或内联编辑 | `Escape` |
| 添加 Todo | 在 `+ 添加待办…` 输入框中输入后按 `Enter` |
| 重命名分组 / 阶段 | 双击名称 |

---

## 命令列表

所有命令通过 `Ctrl+Shift+P` 在 **Check Flow** 分类下可以找到：

| 命令 | 说明 |
|------|------|
| 刷新 | 重新加载 WebView |
| 折叠 / 展开所有分组 | 切换所有分组的折叠状态 |
| 导出为 JSON | 将清单保存为 JSON 文件 |
| 从 JSON 导入 | 从 JSON 文件加载清单 |
| 添加到 Check Flow… | 将 Explorer 中选中的文件添加到指定阶段 |

---

## 数据存储

所有数据保存在 VS Code 的 **工作区状态（workspace state）** 中，每个工作区独立存储，互不干扰。除非你主动导出，否则不会写入任何磁盘文件。

## 配合 AI 使用导入说明

通过面板标题栏 → **导出为 JSON** 将清单导出，然后将文件内容粘贴到 AI 对话中即可。

### 常用提示词

```
以下是我的代码检查清单，请帮我分析哪些文件还没检查，并给出优先级建议：
[粘贴 JSON]
```

```
请根据清单中未完成的文件和 Todo，帮我列出下一步应该重点检查的内容：
[粘贴 JSON]
```

### AI 能帮你做什么

| 场景 | 示例提示词 |
|------|-----------|
| 排列检查优先级 | "哪些未完成的文件最可能存在 Bug？" |
| 自动生成 Todo | "根据文件名，为每个未完成文件生成 3 条检查项" |
| 汇总进度 | "用中文总结各分组的检查进度" |
| AI 编辑后回导入 | 让 AI 返回修改后的 JSON，再导入回插件 |

### JSON 字段说明

```jsonc
{
  "version": 1,               // 固定为 1
  "exportedAt": "ISO-8601",   // 导出时间
  "workspace": "string",      // VS Code 工作区文件夹名称
  "summary": "string",        // 可读的进度摘要
  "groups": [
    {
      "name": "string",       // 分组名称，如"前端检查"
      "phases": [
        {
          "name": "string",   // 阶段名称，如"第一阶段"
          "progress": "x/y files",
          "files": [
            {
              "name": "string",     // 文件名，如 "index.vue"
              "path": "string",     // 磁盘上的绝对路径
              "checked": boolean,   // true = 已检查完成
              "todos": [
                {
                  "text": "string",    // 待办描述
                  "checked": boolean   // true = 该待办已完成
                }
              ]
            }
          ]
        }
      ]
    }
  ]
}
```

### 闭环流程：AI 编辑 → 重新导入

AI 可以返回修改后的 JSON（例如添加了新 Todo、更新了 `checked` 状态或新增了阶段）。应用方式：

1. 将 AI 输出的内容保存为 `.json` 文件
2. 在 VS Code → 面板标题栏 → **从 JSON 导入**
3. 选择 **合并**（追加分组）或 **替换**（覆盖当前数据）
